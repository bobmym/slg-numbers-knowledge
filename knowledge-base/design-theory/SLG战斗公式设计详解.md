## 一、四种战斗公式对比

### 1. 减法伤害公式
```
伤害 = 攻击 - 防御
改良版: 伤害 = 攻击 - (攻击×防御)/(常数+防御)
```
- **适用**：休闲游戏、轻度游戏
- **特点**：数值稳定但容错低

### 2. 乘法伤害公式
```
伤害 = 攻击 × (1 - 防御/(防御+a×等级+b))
```
- **适用**：中度游戏（卡牌、ARPG）
- **特点**：数值稳定，容错高

### 3. 除法伤害公式
```
伤害 = 攻击 × 攻击/(攻击+防御)
```
- **适用**：重度游戏（SLG、MMO）
- **特点**：伤害收益最高，但数值波动大

### 公式特征对比

| 公式 | 数值稳定 | 数值容错 | 综合体验 | 设计难度 | 伤害收益 |
|------|----------|----------|----------|----------|----------|
| 减法伤害 | 较低 | 较低 | 较低 | 较低 | 较低 |
| 减法改良 | 高 | 高 | 高 | 低 | 低 |
| 乘法伤害 | 高 | 高 | 高 | 低 | 低 |
| 除法伤害 | 最低 | 最低 | 最低 | 最高 | 最高 |

> **核心结论**：SLG产品选用除法公式是为了提高ARPPU，通过高烈度战斗方式增加付费。

## 二、SLG战斗公式修正

### 核心问题
SLG中攻击/防御与兵力相关，兵越多攻击与防御越高，但需避免无脑堆兵，需要使用幂函数控制效率。

### 修正后的公式
```
伤害 = 攻击力×兵数^(8/25) × 攻击力×兵数^(8/25) / 
       (攻击力×兵数^(8/25) + 防御力×兵数^(8/25))
```

### 系数选择逻辑
- 使用 `兵数^a` 模式（a=8/25≈0.32）
- 通过兰切斯特模型模拟计算确定系数
- **系数越大**：兵越多效率越高
- **系数越小**：付费内容空间越大
- 必须保证下限有一定斜率，否则基础成长对战斗无影响

### 验证流程
1. 确定攻击、防御、血的上下限
2. 兵分10级，通过兰切斯特平方律计算满足预期层级
3. 设定每级兵的基础系数
4. 观察每级兵提升2倍的梯度曲线
5. 服务端模拟上万场战斗验证公式合理性

## 四、经济系统设计

### 核心公式
```
收获 = 产出率 × 时间
```

### 核心概念
- **盈亏系数**：资源盈亏 = (产出资源 - 消耗资源)/消耗资源
- **r1 = 必要支出/总支出**：决定用户体验

### r1与用户体验对照

| r1值 | 状态 | 追求 | 体验 |
|------|------|------|------|
| 0.6 | 贫困 | 生存 | 压力大，易流失 |
| 0.5-0.6 | 温饱 | 个人成长 | 热情高涨，体验最佳 |
| 0.4-0.5 | 小康 | 社交 | 成长动力下降，体验良好 |
| 0.3-0.4 | 富裕 | 荣誉 | 游戏进入瓶颈 |
| <0.3 | 富豪 | 自我实现 | 迷失，易流失 |

### LTV策略选择
- **提高ARPPU**：战斗收益曲线 > 经济收益曲线（差距越大，ARPPU越高）
- **提高留存/付费率**：经济收益曲线 ≈ 战斗收益曲线（差距越小越好）

## 五、设计流程与复盘

### 完整设计流程
1. **战斗公式选择** → 2. **属性设计** → 3. **属性价值计算** → 4. **经济投放配置** → 5. **复盘验证**

### 复盘三维度
- **经济复盘**：验证产出消耗是否平衡
- **成长复盘**：验证用户成长曲线
- **战斗复盘**：回归最后战斗体验

### 注意事项
- 战斗、经济、成长三模块必须配套
- 确定提高ARPPU后，战斗公式不能选非除法公式
- 一级属性整体价值不能太低，r1需高于0.6
- 赛道选错属于系统性错误，后续无法调整
