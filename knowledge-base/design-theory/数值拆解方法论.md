## 📋 数值拆解的目的

> 不是为了"抄数值"，是为了理解设计意图

| 目的 | 说明 |
|------|------|
| **竞品分析** | 理解竞品设计逻辑，判断自身方向 |
| **方法学习** | 验证已学方法在实际产品中的表现 |
| **方案推导** | 反推公式，复用为自身设计参考 |

---

## 🔄 完整拆解流程

```
目标设定 → 数据收集 → 数据分析 → 假设生成 → 假设验证 → 结论应用
```

### 1. 数据收集

| 方法 | 渠道 |
|------|------|
| 专业网站 | GameRes、GameLook、七麦数据 |
| 玩家论坛 | 玩家实战经验、战报数据 |
| 反编译 | 客户端逆向（高阶，有风险） |
| 同行咨询 | 业内经验参考 |

### 2. 数据分析

从数据中**找规律**，核心关注三点：

- **战斗公式**：是加法型（减法公式）还是乘法型（除法公式）？
- **系统模块价值比**：各系统占整体价值的比例是多少？
- **单模块成长节奏**：成长曲线是线性的还是指数的？

### 3. 假设验证

用收集到的数据**代入验证**：
> 例如：猜测某SLG的伤害公式为 `伤害 = 攻击 × 攻击 / (攻击 + 防御)`
> 用战报数据反推验证

---

## 📐 常见战斗公式类型

| 公式类型 | 特点 | 适用场景 |
|------|------|---------|
| **减法公式** `伤害=攻击-防御` | 数值直观，防御效果强 | 早期RPG，强调单兵 |
| **除法公式** `伤害=攻击²/(攻击+防御)` | 防御边际递减，攻防更均衡 | 主流SLG/RPG |
| **复合公式** `伤害=攻击×技能系数×克制系数` | 最灵活，可多维度调控 | 需要复杂战斗设计 |

**趋势**：近年RPG/卡牌游戏更多使用减法公式，强调单角色体验；SLG因为多单位战斗，除法公式更常见。

---

## 📈 成长曲线设计

| 类型 | 函数 | 特点 |
|------|------|------|
| 线性成长 | `y = kx` | 平稳，中后期乏力 |
| 指数成长 | `y = a^x` | 前期慢，后期爆发 |
| 分段成长 | 分段函数 | 可控制节奏节点 |

**防御属性与减伤关系**：`减伤比例 = X / (aX + b)`（边际递减曲线）

---

## 💡 老六意见

> **方法论是骨架，实际游戏经验是血肉。**
> 
> 拆解的时候不要只盯着数值，要多问"为什么这个设计用这个公式"。
> 比如COK用减法还是除法，背后是对战斗体验和数值膨胀控制的不同考量。
> 
> 建议配合实际拆解案例一起看，光看方法不动手 = 纸上谈兵。
