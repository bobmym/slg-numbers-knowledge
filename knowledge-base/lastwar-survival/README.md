# 【系统篇】核心玩法拆解

## 2.1 核心游戏循环

### 入口层（超休闲外衣）

```
打开游戏
    ↓
约5分钟"跑酷/射击"类超休闲小游戏（与广告素材高度一致）
    ↓
前5分钟体验基本覆盖广告承诺
    ↓
自然过渡到城建/策略界面
```

> **关键设计**：广告展示的"数学射击"玩法是游戏内实际可玩内容的一部分，集成在前几分钟的新手引导中，实现了**广告=游戏体验**的一致性。

### 主循环（城建设定 / 放置成长）

```
建造 / 升级建筑（基地建设）
    ↓
收集资源（金币/材料等）
    ↓
训练部队
    ↓
Frontline（合并战斗模式）推关
    ↓
获得更强装备/资源
    ↓
强化部队
```

### 联盟层（社交爆发）

```
加入联盟
    ↓
联盟活动（僵尸围城 / 联盟战）
    ↓
GVG对抗 → 社交粘性
    ↓
联盟科技 / 共同目标
```

---

## 2.2 Frontline 合并战斗系统 — 核心X要素

这是Last War区别于传统SLG的最核心机制，也是广告素材的核心内容。

### 什么是Frontline？

Frontline是游戏内**独立的合并战斗玩法模式**，在游戏前期独立存在，中后期与城建SLG并行。

### 机制详解

| 要素 | 说明 |
|------|------|
| **战斗形式** | 竖版合并战斗，玩家操作士兵不断推进 |
| **合并机制** | 同类士兵可合并升级，合并后战力大幅提升 |
| **伤害机制** | 广告中展示的"数字相撞相乘"视觉效果 |
| **敌人** | 僵尸大军，种类逐渐增加 |
| **推进方式** | 合并更大的单位，对抗不断涌来的僵尸潮 |
| **放置属性** | 合并后单位自动攻击（近似放置类）|

### 广告素材与实际游戏的对应关系

| 广告展示 | 实际游戏内容 |
|----------|------------|
| 士兵越打越多，数字相乘 | Frontline合并机制 |
| 数学计算式视觉 | 合并时数字显示 |
| 激烈僵尸战斗 | Frontline关卡战斗 |
| 超休闲爽感 | Frontline前几关体验 |

> **重要说明**：合并机制**仅存在于Frontline玩法**，不直接混入城建设定部分。这是玩家可能产生"被骗"感的来源，但同时也解释了为何广告能精准转化——因为广告展示的玩法确实在游戏里存在。

### 设计目的

| 维度 | 分析 |
|------|------|
| **获客** | 超休闲广告吸引泛用户导入 |
| **留存** | 前5分钟爽感强，正反馈密度极高 |
| **过渡** | 作为新手引导，让SLG深度不那么"沉" |
| **差异化** | 与Whiteout Survival的模拟经营形成鲜明区隔 |

> **数值策划参考**：放置合并的爽感来自**即时正反馈**（合并=立即变强），与SLG的长周期养成形成互补。前者满足"立即爽"，后者满足"慢慢变强"。

## 2.3 基地建设系统

### 基础逻辑

基地建设是游戏的**核心资源产出层**，也是向传统4X转化的关键：

| 建筑类型 | 功能 |
|----------|------|
| 总部（HQ）| 核心建筑，决定整体发展上限 |
| 资源建筑 | 产出金币、材料等 |
| 训练建筑 | 训练各类部队（T1~T10）|
| 防御建筑 | 防守基地 |
| 联盟相关建筑 | 解锁联盟科技 |

### HQ升级策略（🟠社区整理）

- HQ是**最优先升级**的建筑，决定解锁内容
- 训练建筑升级到特定等级后，才可训练更高级兵种
- 兵种升级到T9或T10后，再进行晋升（效率最优）

### 与Whiteout的差异化

| 维度 | Last War | Whiteout Survival |
|------|----------|-------------------|
| 建筑数量 | 较少，路径清晰 | 极简，单机体验 |
| 升级反馈 | 中等密度 | 高密度（模拟经营节奏）|
| 资源压力 | 中等 | 极低（几乎无卡点）|

## 2.4 兵种系统

### 兵种等级梯度

游戏内兵种从T1到T10分级，**不提前晋升是核心策略**（🟠社区共识）：

| 阶段 | 策略 |
|------|------|
| T1-T9 | 训练积累，不晋升 |
| T10 | 晋升节点（消耗大，质变也最大）|
| 晋升后 | 战力大幅提升 |

> **核心建议**：在解锁10级兵种之前不要晋升。晋升需要更多的时间和资源。

### 战力提升优先级（🟠Reddit玩家社区）

根据Reddit玩家投票整理，战力提升优先序：

1. **装备升级**（影响最大）
2. **HQ升级**（解锁内容）
3. **兵种训练**（T10晋升节点）
4. **科技研究**
5. 英雄强化

> **数值策划参考**：Last War的战力系统中，**装备是核心变量**，等级/兵种的边际收益相对平稳。这与Whiteout以"英雄星级"为核心变量的设计形成对比——两者都是"一个核心杠杆"，但具体杠杆位置不同。

---

## 2.5 英雄系统

### 定位

英雄在Last War中是**战力强化的重要路径之一**，但与Whiteout的"英雄即核心"相比，Last War的英雄定位更接近**战力辅助层**（装备 > 英雄）。

### 英雄类型

| 类型 | 定位 |
|------|------|
| 攻击型 | 输出伤害 |
| 防御型 | 吸收伤害 / 守护 |
| 支援型 | 加速/治疗等辅助 |

### 英雄成长路径

```
获取英雄 → 升级（等级） → 装备强化 → 科技配合
```

### 战力贡献占比

根据社区玩家经验：**装备 > 等级 ≈ 科技 > 英雄**

> **数值策划参考**：Last War的付费深度分布在**装备系统**上（可以通过大量IAP直接强化），而Whiteout的付费深度更多压在**英雄获取（抽卡）**上。两者都是"一个核心付费竖井"，但竖井所在系统不同。

## 2.6 联盟系统

### 核心功能

| 功能 | 说明 |
|------|------|
| 联盟捐献 | 获取联盟货币 |
| 联盟科技 | 解锁属性加成（攻击/防御/资源等）|
| 联盟活动 | 僵尸围城 / 联盟战 |
| 资源共享 | 成员间互助 |

### 设计特点

- 联盟活跃度影响活动参与资格（需在活跃联盟才可参加活动）
- 联盟贡献有**个人上限**（防止垄断）
- 有联盟指挥价值（协调进攻路线、资源分配）

# 【数值篇】数值设计

## 3.1 资源体系

### 主要资源类型

| 资源 | 功能 | 特性 |
|------|------|------|
| 金币 | 基础货币，建筑/兵种/装备 | 最通用的资源 |
| 木材/钢材 | 城建消耗 | 有一定稀缺性 |
| 钻石 | 高级货币，加速/召唤 | 核心IAP货币 |
| 部队（兵种）| 战力基础 | 有T1-T10梯度 |
| 装备 | 战力核心杠杆 | 可强化/升级 |
| 联盟货币 | 联盟科技 | 捐献产出 |

### 资源产出设计理念

> Last War的资源压力**中等**，既不像传统SLG那样紧张（负循环），也不像Whiteout那样极度充裕。设计上保持了适度的"资源饥渴"，以驱动活动参与和加速道具消耗。

---

## 3.2 战力数值框架

### 战力构成

```
总战力 = 装备战力 + 兵种战力 + 英雄战力 + 科技战力 + 联盟科技加成
```

### 战力提升优先级（🟠Reddit玩家社区共识）

| 优先级 | 升级项 | 重要性原因 |
|--------|--------|------------|
| #1 | 装备强化 | 边际收益最高 |
| #2 | HQ升级 | 解锁核心内容 |
| #3 | 兵种晋升 | T10节点质变 |
| #4 | 科技研究 | 全局加成 |
| #5 | 英雄培养 | 辅助增益 |

> **数值策划参考**：Last War的数值框架是**装备驱动的发散模型**——装备强化没有硬上限（可反复刷），为付费用户提供了近乎无限的深度。这与Whiteout的"英雄星级天花板"形成对比。

---

## 3.3 兵种数值梯度

### T1~T10 兵种曲线

| 兵种等级 | 解锁条件 | 质变点 | 策略建议 |
|----------|----------|--------|----------|
| T1-T4 | HQ低等级 | 无特殊 | 过渡用 |
| T5-T8 | 中期兵种 | 中等提升 | 积累阶段 |
| T9 | 高阶兵种 | 战斗力明显提升 | 不晋升，继续积累 |
| T10 | 最高阶 | **最大质变节点** | 晋升 |

> **核心设计原则**：T10是兵种的最大质变节点，晋升消耗巨大但收益最大。这形成了"积累-T10-质变"的核心爽感周期。

---

## 3.4 战斗数值设计

### 伤害机制

基于Frontline和城建战斗的**双轨战斗设计**：

| 战斗类型 | 机制 | 爽感来源 |
|----------|------|----------|
| Frontline战斗 | 放置自动攻击 + 合并放大 | 合并数字放大，立即可见 |
| 基地战斗 | 手动进攻/防守 | 策略配合 |

### 与Whiteout的数值对比

| 维度 | Last War | Whiteout Survival |
|------|----------|-------------------|
| 爽感来源 | 合并数字倍增 | 英雄强化/天赋解锁 |
| 战斗节奏 | Frontline快节奏 | 回合制/PVP节奏较慢 |
| 战力差距 | 装备差距主导 | 英雄星级差距主导 |
| 策略深度 | 合并策略 + 兵种搭配 | 英雄协同 + 职业克制 |

## 4.2 重点活动拆解

---

### 🔵 活动一：僵尸围城（Zombie Siege）

| 波次进攻 | 僵尸潮逐波进攻 |

| 奖励类型 | 说明 |
|----------|------|

| **频率高** | 定期活动形成登录锚点 |

### 🔵 活动二：联盟战（Alliance Battle）

**概述**：联盟之间的GVG对抗，是游戏最核心的对抗性活动。

#### 基本信息

| 字段 | 内容 |
|------|------|
| 周期 | 定期 |
| 参与 | 联盟 vs 联盟 |
| 形式 | 进攻/防守积分制 |

#### 机制特点

- 进攻方和防守方都有积分
- 联盟成员贡献度决定整体排名
- 指挥协调能力影响胜负
- 赛季制：每次联盟战构成赛季的一部分

#### 设计目的

| 维度 | 分析 |
|------|------|
| **社交深度** | GVG比PVE带来更强的联盟认同感 |
| **付费驱动** | 大战前强化装备/部队是典型付费场景 |
| **指挥价值** | 联盟需要指挥者，提升核心玩家粘性 |
| **赛季奖励** | 长期目标，赛季积分与最终奖励挂钩 |

### 🔵 活动四：限时礼包（Live Ops）

**概述**：以紧迫感为驱动的付费转化活动。

#### 机制设计

| 策略 | 说明 |
|------|------|
| 时间限制 | 限时商店，倒计时紧迫感 |
| 首充优惠 | $0.99/$1.99 低价钩子 |
| 阶梯礼包 | 逐级提升，价值感递进 |
| 新英雄up | 新英雄概率提升 |
| 节日特惠 | 农历新年/情人节等节点 |

#### 付费设计哲学

> Last War的付费设计是**渐进式深度**：用极低价格（如$0.99）让用户完成首次付费，之后在正确的时间节点推送更高价格商品，配合限时紧迫感促成转化。

## 4.3 活动日历排期规律

基于公开信息，Last War的活动遵循以下节奏规律：

```
每周：
  - 僵尸围城 × 多次
  - 联盟战 × 1-2次

每月：
  - 赛季结算
  - 新赛季开启
  - 重大版本更新

节点性：
  - 农历新年（大规模活动）
  - 情人节（限定优惠）
  - 其他节日节点
```

> **数值策划参考**：Last War的活动节奏与Whiteout类似（日有PVE锚点 + 周有GVG + 月有大事件），但Last War的**付费活动密度更高**（限时礼包频率更高），这与其ARPDAU更高的数据相吻合。
