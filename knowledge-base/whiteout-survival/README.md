# 【系统篇】核心玩法拆解

## 2.1 核心游戏循环

无尽冬日的核心设计理念是**「用模拟经营的外壳承接用户，用SLG的深度留住用户」**。

### 主循环（模拟经营阶段 / 单机闭环）

```
采集资源（自动产出）
    ↓
建造 / 升级建筑
    ↓
解锁新功能 / 推进温度线
    ↓
英雄获得经验 / 强化英雄
    ↓
冒险远征推图（PVE）
    ↓
更多资源产出 / 更强英雄
```

这一阶段**没有掠夺、没有战争**，纯粹是放置+强化的单机体验。设计目的：**消除传统SLG前30分钟的挫败感，用"温暖的成长感"留住用户**。

### 长循环（SLG阶段 / 社交爆发）

```
英雄强化 → 战力提升
    ↓
联盟活动（猎熊 / 兵工厂 / 堡垒战 / SVS）
    ↓
GVG对抗 → 社交粘性
    ↓
大地图 / 跨服战
    ↓
持续驻留 / 付费深化
```

---

## 2.2 温度系统 — 核心生存压力机制

温度是贯穿模拟经营阶段的核心机制，带来了**与其他SLG本质性的差异**。

### 设计逻辑

- 地图存在**温度线**（随主线推进，气温越来越低）
- 玩家的基地需要维持**供暖**，否则建筑效率下降甚至冻结
- 暴风雪等天气事件会**周期性逼近**，制造紧迫感

### 设计目的

| 效果 | 说明 |
|------|------|
| **代入感** | 末日冰雪题材氛围营造 |
| **节奏控制** | 温度线自然引导玩家推进主线 |
| **资源压力** | 供暖消耗创造资源管理的策略空间 |
| **内容揭示** | 随着温度下降，解锁新的剧情和内容 |

> **数值策划参考**：温度系统本质上是**隐形的时间阀门**——不强迫玩家付费，但会放大落后感，从而间接驱动付费意愿。

---

## 2.3 城建系统 — 双轨设计

根据知乎文章（2024年2月）描述，无尽冬日在城建设计上做了**与传统COK-Like的显著区分**：

### 传统COK-Like城建的痛点
- 建筑数量多（20+建筑）
- 单个建筑升级周期长（可达数周）
- 玩家辛苦一个月才感受一次"质变"
- 大量建筑之间互相依赖，形成"卡点墙"

### 无尽冬日的双轨设计

**养成建筑（对应模拟经营玩法）**
- 放置即可产出资源
- 主要服务于英雄强化
- 升级路径线性，成长感密集

**SLG建筑（对应对抗玩法）**
- 部队训练、防御工事等
- 较少受卡线限制
- 解锁后对接联盟GVG内容

### 核心理念

> **"以英雄为核心养成方向，城建上弱化双循环规则，让用户的精力从城建转移到英雄与大地图玩法。"**
> — 知乎用户对无尽冬日的分析

## 2.4 英雄系统

英雄是游戏**最重要的战力载体**，也是最核心的付费深度。

### 英雄获取
- 通过游戏内活动/召唤获取
- 有品质梯度（部分英雄通过氪金活动 exclusivity 获取）

### 英雄强化路径

| 维度 | 说明 |
|------|------|
| **等级** | 通过冒险远征获得经验 |
| **星级** | 消耗同名卡提升，星级解锁天赋属性 |
| **装备** | 专属装备（数值装备，有专精系统）|
| **协同技能** | 多英雄组合激活协同效果（核心策略点）|

### 英雄天赋（2025年版本更新）
- 英雄星级提升可**永久获得属性加成**（天赋属性）
- 这强化了"英雄是资产"的核心认知

### 英雄协同系统

这是游戏的核心策略层：
- 不同英雄组合激活协同效果
- 战斗中阵型站位影响协同触发
- 形成了类似"职业克制 + 组合增益"的战术深度

### 数值策划参考

英雄作为游戏内**最强力的战力载体**，其数值曲线往往远陡峭于建筑或兵种。这意味着：
- 英雄等级差距对战力影响极大
- 英雄相关活动（如召唤概率up）是核心付费驱动
- 英雄迭代（推出新英雄）是版本更新的主要商业化手段

## 2.5 兵种系统

### 基础设定
- 游戏内有**多种兵种**，分不同等级
- 兵种通过训练建筑产出
- **10级前不建议晋升**（晋升消耗资源大，建议攒到10级再进行，Reddit玩家社区共识）

### 兵种升级策略

| 阶段 | 策略 |
|------|------|
| 低本 | 训练基础兵种，快速提升战力 |
| 中本 | 升级训练建筑，解锁高级兵种 |
| 高本 | 10级后晋升，享受效率最优解 |

> 🟠 **来源**：Reddit r/whiteoutsurvival 玩家整理 🟠

---

## 2.6 联盟系统

### 核心功能
- **资源共享**：成员间互助加速建造
- **联盟技能**：贡献度解锁联盟科技
- **活动参与**：猎熊、兵工厂、堡垒战等GVG活动
- **R4自荐机制**（2025年9月更新）：联盟活跃R4人数不足时开启自荐

### 联盟活动生态

| 活动 | 周期 | 类型 |
|------|------|------|
| 猎熊行动 | 每日（UTC 13:00-15:00，两次）| PVE集结 |
| 兵工厂争夺战 | 每周 | GVG占点 |
| 堡垒争夺战 | 周末 | GVG据点 |
| SVS（跨服战）| 每月 | 超大GVG |

## 2.7 冒险远征系统

- **核心PVE内容**，玩家推图获取经验和资源
- 是英雄经验的主要来源
- 设计为**完全单人**，与联盟无关
- 是模拟经营阶段向SLG阶段的自然过渡

# 【数值篇】数值设计

> **声明**：以下数值基于社区整理与公开报道，完整精确数值表需通过实际游戏数据抓取验证。部分数据为玩家经验分享，标注🟠。

## 3.1 资源体系

### 主要资源类型

| 资源 | 功能 | 产出方式 |
|------|------|----------|
| 木材 | 基础城建资源 | 伐木场（自动产出）|
| 钢材 | 高级城建资源 | 钢铁厂（自动产出）|
| 电力 | 驱动特定建筑 | 发电厂（自动产出）|
| 温暖值 | 核心特色资源，维持温度 | 熔炉产出 |
| 钻石 | 加速/召唤/商城货币 | 任务/活动/付费 |
| 加速道具 | 加速建造/训练 | 活动奖励 |
| 领主体力 | 参与特定活动 | 日常/活动发放 |

### 产出设计理念

> **核心原则：资源大量产出，不过度卡点**
> 
> 区别于传统COK-Like的"采集-消耗"负循环，无尽冬日让资源处于**相对充裕**的状态，玩家不会感受到"缺资源"的焦虑。这是刻意设计——**把焦虑释放，转移到英雄养成和GVG上**。

---

## 3.2 城建数值框架

### 熔炉系统（Furnace）

熔炉是游戏内最核心的产出建筑，其设计特点：

| 特点 | 说明 |
|------|------|
| 加速升级 | 适合长时间升级或同时升级多个建筑 |
| 效率杠杆 | 熔炉等级越高，整体城建效率越高 |
| 策略点 | 何时使用熔炉加速是重要的资源分配决策 |

> 🟠 **来源**：Thiria Games 白茫茫生存炉指南 🟠

### 城建升级曲线（规律性描述）

基于社区经验：
- 前期建筑升级时间**分钟级~小时级**，快速给正反馈
- 中期进入**小时~天级**，成长感放慢但仍有密度
- 后期向**天~周级**过渡，此时英雄成为主要战力来源

> **数值策划参考**：这种"前期密-中期稳-后期长"的曲线是SLG的典型节奏。但无尽冬日通过英雄系统的存在，让玩家在城建放缓期仍有**明确的战力增长路径**——这是其留住用户的关键。

---

## 3.3 英雄数值框架

### 英雄数值成长模式

```
星级（SSR/SR/R卡）× 等级（经验获取）× 装备等级 = 总战力
```

| 维度 | 成长速度 | 付费耦合度 |
|------|----------|------------|
| 等级（经验）| 中（放置产出，相对公平）| 低 |
| 星级（升星）| 慢（依赖同名卡获取）| 高（核心付费深度）|
| 装备 | 中 | 中 |
| 天赋（星级解锁）| 极慢（随星级永久附加）| 高 |

### 品质梯度设计

游戏内英雄有明确的品质分层（SSR > SR > R），高品质英雄：
- 基础属性更高
- 协同效果更强
- **获取门槛更高**（部分限氪）

> **数值策划参考**：品质梯度是最典型的"战力差"设计——付费用户在数值上拉开差距，同时通过"协同系统"制造阵容策略深度，让品质差距不是简单的线性叠加，而是有策略博弈空间。

---

## 3.4 战斗数值

### 属性克制

基于英雄职业分工（盾/矛/弓等），存在**职业定位克制关系**：

| 职业 | 定位 | 克制关系 |
|------|------|----------|
| 盾兵（如谢尔盖）| 前排肉盾 | 吸收伤害 |
| 弓兵（如巴西提）| 后排输出 | 输出位 |
| 矛兵（如茉莉）| 突击 | 中距离切入 |

### 英雄队长机制

特定PVE活动（如猎熊）中，玩家需要选英雄作为**队长**带队参与，英雄强弱直接影响活动得分。

> **数值策划参考**：英雄队长机制将英雄强化数值**直接转化为活动竞争力**，是连接"英雄养成"与"联盟活动"的核心枢纽。

## 4.2 重点活动拆解

---

### 🔵 活动一：猎熊行动（Bear Hunt）

**概述**：每日联盟PVE集结活动，是最轻量、最高频的联盟活动。

#### 基本信息

| 字段 | 内容 |
|------|------|
| 周期 | 每日，UTC 13:00-15:00（两次）|
| 参与门槛 | 联盟成员，在活动期间进入 |
| 活动时长 | 约2小时一局 |

#### 参与方式

1. 联盟成员进入猎熊战场
2. 选取英雄（英雄作为**队长**）带队参与
3. 对暴怒巨熊进行**集结攻击**
4. 根据对熊造成的伤害量分配奖励

#### 数值结构（🟠社区来源，准确性待验证）

| 行为 | 奖励 |
|------|------|
| 完成1次击杀（前3次）| 金币x40（悬赏大厅使用）|
| 完成1次击杀 | 领主体力x5、1小时通用加速x3、100钻石x5 |
| 完成3次击杀 | VIP奖励x2、1小时通用加速 |

#### 设计目的

| 维度 | 分析 |
|------|------|
| **促活** | 每日两次，形成稳定登录锚点 |
| **社交** | 联盟成员共同参与，增强归属感 |
| **战力验证** | 英雄强弱直接转化为伤害，刺激英雄培养需求 |
| **付费钩子** | 伤害排名靠后的玩家有强化英雄的动机 |

#### 英雄推荐（🟠社区整理）

- **前期万金油**：谢尔盖（盾）+ 茉莉（矛）+ 巴西提（弓）
- 高氪玩家可使用尼莫或熊妹

---

### 🔵 活动二：兵工厂争夺战

**概述**：每周联盟GVG占点活动，需要联盟成员配合占领战略据点。

#### 基本信息

| 字段 | 内容 |
|------|------|
| 周期 | 每周（周内工作日）|
| 参与门槛 | 活跃联盟成员 |
| 活动类型 | GVG据点占领 |

#### 战场机制

战场内有以下关键建筑据点需要占领：

| 据点 | 优先级 | 说明 |
|------|--------|------|
| 中转站 | 最高 | 核心战略位置 |
| 军火营 | 高 | 积分来源 |
| 雇佣兵营地 | 中 | 辅助功能 |
| 蒸汽锅炉 | 低 | 辅助功能 |

#### 军工积分获取策略

1. **优先占领中转站**（积分权重最高）
2. 配合联盟指挥，统一进攻节奏
3. 防御方设置陷阱（联盟陷阱）

#### 设计目的

| 维度 | 分析 |
|------|------|
| **策略深度** | 多据点分配需要联盟策略（而非无脑堆伤害）|
| **社交粘性** | 联盟协同作战强化成员关系 |
| **指挥价值** | 有指挥的联盟 vs 无指挥联盟形成明显差距 |
| **陷阱系统** | 增加反制空间，提升策略博弈 |

> **数值策划参考**：兵工厂争夺战的**积分系统**是多维度的——不是只看总伤害，而是看占领质量。这给不同战力层次的玩家提供了不同的贡献方式（占点 / 防守 / 辅助）。

---

### 🔵 活动三：堡垒争夺战（Fortress Battle）

**概述**：周末GVG据点攻防战，比兵工厂规模更大。

#### 基本信息

| 字段 | 内容 |
|------|------|
| 周期 | 周末 |
| 规模 | 大于兵工厂 |
| 参与 | 联盟成员协同 |

#### 机制特点

- 联盟成员可以设置**堡垒陷阱**（需要 strategize 布置）
- 活动结算前5秒**不能拆除联盟陷阱**（增加结算期变数）
- 联盟成员的堡垒战**贡献值**会显示在战报中
- 奖励按贡献值分配

#### 设计亮点

> 堡垒战贡献值的显性化，让**每个成员的努力都能被看见**，这是联盟持续活跃的重要激励机制。

### 🔵 活动四：SVS（跨服战 / Super VS）

**概述**：每月一次的超大规模跨服GVG，是游戏版本级别的核心事件。

#### 基本信息

| 字段 | 内容 |
|------|------|
| 周期 | 每月 |
| 规模 | 跨服，多联盟参与 |
| 时长 | 通常持续较长（玩家社区反映打到半夜）|

#### 阶段设计

根据社区反馈，SVS通常包含以下阶段：

| 阶段 | 内容 |
|------|------|
| 备战阶段 | 联盟集结，战略准备 |
| 正式战斗 | 联盟间大规模对抗 |
| 结算阶段 | 排名奖励发放 |

#### 设计目的

| 维度 | 分析 |
|------|------|
| **版本级事件** | 每月一次，形成大型内容消费周期 |
| **付费爆发点** | 大战前联盟成员有强烈强化战力的需求 |
| **联盟分化** | 服务器实力格局重新洗牌 |
| **社交高潮** | 是游戏内最激烈的社交体验 |

## 4.3 活动日历排期规律

基于公开信息整理，无尽冬日活动排期存在以下规律：

```
每日：
  - 猎熊行动 × 2（UTC 13:00 / UTC 13:00-15:00滚动）

每周：
  - 兵工厂争夺战（周内）
  - 堡垒争夺战（周末）

每月：
  - SVS（跨服战）1次

季节性：
  - 周年庆（连续登录7天，1000张盛典礼券等）
  - 冬季/节日主题限时活动
```

> **数值策划参考**：活动的节奏设计遵循**"日有锚点，周有对抗，月有大事件"**的三层结构。日活动留住用户，周活动制造中等付费窗口，月活动创造收入峰值。

## 4.4 活动的数值共性模式

通过以上活动分析，可以提炼以下数值设计共性：

### 奖励结构

| 奖励类型 | 作用 | 典型场景 |
|----------|------|----------|
| 钻石 | 免费货币，维护免费玩家 | 猎熊击杀奖励 |
| 加速道具 | 压缩等待时间 | 各类活动通用 |
| 领主体力 | 参与凭证 | 猎熊 / 活动任务 |
| VIP点数 | 身份/功能解锁 | 击杀奖励 |
| 联盟贡献 | 联盟科技 | 联盟活动参与 |

### 消耗结构

| 消耗类型 | 作用 |
|----------|------|
| 领主体力 | 参与门槛控制 |
| 时间（等待）| 形成活动周期节奏 |
| 战力（英雄/兵种）| 参与结果的分层 |
